# Release History

Ceedling's release history is documented in three files that live in the
repository and follow [keepachangelog.com](https://keepachangelog.com/en/1.0.0/)
conventions. These documents span the full, cumulative project history and are
maintained as standalone files rather than being versioned with the documentation
bundle.

## Release Notes

**[ReleaseNotes.md](https://github.com/ThrowTheSwitch/Ceedling/blob/main/docs/ReleaseNotes.md)**

Detailed notes for each release covering new features, behavioral changes,
bug fixes, and known issues. This is the primary reference for understanding
what changed between versions.

## Breaking Changes

**[BreakingChanges.md](https://github.com/ThrowTheSwitch/Ceedling/blob/main/docs/BreakingChanges.md)**

A focused list of breaking changes across releases. Consult this document when
upgrading to understand what in your project configuration or test code may need
to be updated.

## Changelog

**[Changelog.md](https://github.com/ThrowTheSwitch/Ceedling/blob/main/docs/Changelog.md)**

A concise, chronological log of all changes following the keepachangelog.com
format. Useful for a quick scan of what changed in any given release.
